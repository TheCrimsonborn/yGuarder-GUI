Core Capabilities

Everything You Need to Protect Your Java IP

Built for security engineers, DevSecOps teams, and developers who take code protection seriously.

Bytecode-Level Obfuscation

Powered by yGuard 5.0.0, the industry-trusted bytecode manipulation engine. Rename packages, classes, methods, and fields with advanced naming schemes including mix-ascii and small.

yGuard 5 Engine

Zero-Dependency Portable

Ships with a bundled Azul Zulu JRE 21 FX runtime. Double-click run.bat and go. No Java installation required on the host machine.

Portable

Granular Keep Rules

ASM 9.6-powered JAR introspection lets you control protection at the package, class, method, and field level — all from an interactive CheckBox tree view.

ASM 9.6

Modern Dark UI

Built with JavaFX 21 and AtlantaFX skin for a premium, pixel-perfect interface. Supports Turkish and English with dynamic language switching.

JavaFX 21 · AtlantaFX

Docker & CI/CD Ready

Multi-stage Dockerfile, GitHub Actions workflows, and a GHCR-based image publishing pipeline included out of the box. Ship protected artifacts automatically.

DevSecOps

Attribute & String Protection

Selectively strip or obfuscate debug attributes (LineNumberTable, SourceFile, LocalVariables) and class name strings to harden against reflection-based attacks.

Hardening
Workflow

From Source JAR to Protected Binary in 3 Steps

01

Load Your JAR

Select your compiled JAR file. yGuarder instantly analyzes its bytecode using ASM 9.6 and displays all packages, classes, methods, and fields in a navigable tree.

→
02

Configure Protection

Select which packages and classes to keep, choose a naming scheme, define external library dependencies, and configure attribute stripping options.

→
03

Obfuscate & Ship

Hit Obfuscate. The yGuard engine processes the bytecode and outputs a hardened JAR. The log console shows real-time progress and output path.

DevSecOps

Integrate Into Any Pipeline

yGuarder isn't just a desktop tool. Its Bridge Architecture cleanly separates the obfuscation engine from the UI, making it trivial to embed in automated workflows.

Docker Multi-Stage Build
Maven build stage + minimal eclipse-temurin runtime. Published to GHCR automatically.
GitHub Actions Workflows
Pre-built workflows for Docker publishing and release packaging on every push.
Qodana Code Quality
Integrated Qodana analysis with custom inspection profiles for ongoing code quality monitoring.
SECURITY.md & CVE Policy
Formal security policy, responsible disclosure process, and dependency CVE tracking built-in.

Quick Start

Up and Running in Minutes

1
Download the portable package
yGuarder-Portable-v1.0.0.zip Download →
2
Install the bundled Azul Zulu JRE 21 FX runtime
PowerShell 
.\install_runtime.ps1
3
Launch the application
CMD / Explorer 
run.bat
Terminal 
# Pull the pre-built image from GHCR
docker pull ghcr.io/thecrimsonborn/yguarder-gui:edge

# Or build & run locally with docker-compose
git clone https://github.com/TheCrimsonborn/yGuarder-GUI.git
cd yGuarder-GUI
docker-compose up --build
PowerShell / Bash 
# 1. Clone the repository
git clone https://github.com/TheCrimsonborn/yGuarder-GUI.git
cd yGuarder-GUI

# 2. Download all dependencies to ./lib
.\download_deps.ps1

# 3. Build the fat JAR
mvn clean package

# 4. Run (requires JavaFX-compatible JRE)
java -jar target/yguarder-gui-1.0-SNAPSHOT-all.jar
